# 水星引力m Markdown 文章模板

4 套模板，覆盖全部内容板块。使用规则见 skill md-article-templates。

## 模板一：技术教程 (tech / notes / science)

```
---
title: "文章标题"
date: 2026-07-12 01:00:00
categories:
  - tech
tags:
  - 标签1
  - 标签2
description: "一句话摘要，120字以内。"
---

> 一句话概括本文解决的问题和核心结论。

## 前言

为什么写这篇、背景是什么、读者能学到什么。

## 核心方案

### 方案概览

**一句话**描述方案核心思路。

关键数据：
- 指标 A：优化前 X → 优化后 Y
- 指标 B：优化前 X → 优化后 Y

### 方案一：xxx

具体操作步骤：

1. **第一步** — 做什么
   ```bash
   # 命令
   ```
2. **第二步** — 做什么
   ```python
   # 代码
   ```
3. **第三步** — 做什么

> **注意**：常见坑。

## 踩坑记录

### 坑 1：xxx

**现象**：发生了什么
**原因**：为什么
**解决**：怎么修的

## 总结

| 维度 | 结论 |
|------|------|
| 效果 | xxx |
| 成本 | xxx |
| 推荐 | ✅ / ⚠️ / ❌ |

几句反思或建议，像跟朋友聊天一样结束。
```

## 模板二：动漫推荐/解析 (anime)

```
---
title: "文章标题"
date: 2026-07-12 01:00:00
categories:
  - anime
tags:
  - 番剧推荐
  - 标签2
description: "一句话吸引人的摘要。"
---

> 一句能概括整篇文章吸引力的话。

## 为什么聊这部

个人怎么接触到这部作品的、第一印象、为什么值得写一篇。

## 故事梗概（无剧透版）

**世界观**：xxx
**主线**：xxx
**看点**：xxx

> 「一句剧中经典台词或弹幕热评」

## 角色群像

### 角色 A

**身份**：xxx
**声优**：xxx
**印象**：几句话描述角色魅力和成长弧线

## 亮点分析

### 剧情结构

- **优点**：节奏、反转、埋伏笔
- **槽点**：哪里节奏慢了、哪里逻辑硬伤

### 作画与演出

**制作公司**：xxx
**关键集数**：第 X 集（名场面描述）
**音乐**：OP/ED 评价，推荐曲目

## 适合人群

- ✅ 喜欢 xxx 类型的
- ✅ 能接受 xxx 元素的
- ❌ 对 xxx 敏感的慎入

## 评分

| 维度 | 评分 | 说明 |
|------|------|------|
| 剧情 | ⭐⭐⭐⭐ | 节奏好但中期略拖 |
| 作画 | ⭐⭐⭐⭐⭐ | 打戏帧数拉满 |
| 音乐 | ⭐⭐⭐⭐ | OP 上头，BGM 适配 |
| 总评 | ⭐⭐⭐⭐ | 值得追 |

最后一句安利，像发朋友圈一样收尾。
```
