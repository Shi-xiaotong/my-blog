今天没有重磅的发布或政策，但开发者社区的一股情绪正在发酵。GitHub Copilot 的按量计费正式生效一个周期后，账单的数字让很多人坐不住了。这事儿的连锁反应可能比想象中大得多——它正在改变一个基本假设：AI 到底应该是固定成本还是可变成本。
## Copilot 按量计费首周账单出炉：从 $29 到 $3000 的跨越

<!-- more -->

GitHub Copilot 切换到按量计费后，第一周的账单数据开始在社交媒体上流传。最引人注目的案例来自一位在 Reddit 上发帖的全栈开发者：他使用 Copilot 进行日常开发，上个月还是 $29 的 Pro 订阅费，本月预估费用达到了 $750。更夸张的是，一位重度使用 agent 会话的开发者晒出了接近 $3000 的月度预计。

GitHub 的产品逻辑其实不难理解：在 agent 模式下，一次会话可能需要调用数十次甚至上百次模型推理，消耗的 token 量远超传统的代码补全。按 $0.01 一个 Credit 计算，一次 agent 会话就要消耗 $30-40 的 Credit。对于每天进行数十次 agent 会话的团队来说，这个数字迅速膨胀。

但开发者们的不满并不完全来自价格本身——而是来自"预期断裂"。过去几年，AI 工具一直给人一种"固定月费无限用"的错觉。GitHub 的这一刀切断了这种预期，也意味着整个行业正在从"补贴获客"模式转向"价值定价"模式。Cursor 的 CEO 在社交媒体上含蓄地表达了竞争立场："我们的定价保持可预测。"Claude Code 则强调其 Pro 订阅没有 token 计量。

更深层的影响可能在于：如果 AI 编程助手从固定成本变成可变成本，企业的预算模型和开发团队的编码习惯都需要重新调整。对于 high agency 的高效率开发者来说，这可能不是大问题——他们本来就在用工具创造远高于成本的价值。但对于轻度使用者来说，这次涨价可能会让他们重新考虑是否值得付费。
