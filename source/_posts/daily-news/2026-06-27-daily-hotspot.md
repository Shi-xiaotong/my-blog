今天，Anthropic 将一场知识产权纠纷摆上了台面。该公司公开指控阿里巴巴对其 Claude 模型进行了大规模蒸馏攻击——总计达 2880 万次定向 Prompt 调用，用于训练其 Qwen 系列模型。这是 AI 行业中关于"模型蒸馏"最严重的公开指控。
## Anthropic 指控阿里巴巴进行 2880 万次蒸馏攻击

<!-- more -->

Anthropic 今天发布了一份详细的调查报告，指控阿里巴巴在未经授权的情况下，通过逆向调用 Claude API 的方式，进行了大规模模型蒸馏（Distillation）。报告称，攻击涉及 2880 万次定向 Prompt 调用，目的是提取 Claude 的能力用于训练阿里巴巴的 Qwen 系列模型。

Anthropic 的安全团队在报告中详细列出了证据：异常的 API 调用模式、来自特定 IP 范围的持续访问请求、以及在某些测试中 Qwen 模型在某些场景下会"误将自称识别为 Claude"——这被认为是蒸馏过程中留下的痕迹。

蒸馏在技术上并不违法，许多 AI 公司（包括美国的头部公司）也在使用这项技术。问题在于，以每秒数万次的速度持续调用竞争对手的付费 API 来训练自己的模型，这明显违反了 Anthropic 的服务条款。Anthropic 的 CTO 表示，这次攻击"是我们迄今见过的规模最大、系统化程度最高的蒸馏攻击"。

Alibaba 对此的回应较为克制。当天晚些时候，阿里巴巴发布内部通知，禁止员工使用 Claude Code 进行工作，理由是"Claude Code 被发现存在后门风险"。这份通知被南华早报获取并报道。Alibaba 没有直接回应蒸馏的指控，但"禁止使用"的决定在某种意义上印证了事态的严重性。
